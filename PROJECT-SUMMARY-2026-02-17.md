# Fate & Feeling 작업 요약 (2026-02-17)

## 핵심 리팩토링
- 모바일 우선 네비게이션 재설계
  - 하단 바를 2줄/혼합 구조에서 **1줄 5메뉴**로 통일
  - 순서: 홈 / 비책 / 상담 / 총운 / 보관함
  - active 상태는 굵은 테두리 제거, 골드 컬러 강조로 단순화
- 상단 우측 햄버거(☰) 동작 통일
  - 고정 링크 제거, 공통 **빠른 이동 메뉴(바텀 시트)**로 통일
  - 닫기 버튼/배경 클릭/ESC 닫기 처리

## 전체총운/리포트 흐름
- 전체총운 입력 후 확인 섹션 자동 스크롤 구현
  - 2단 스크롤 제거, 단일 deterministic scroll로 안정화
- 기존 사용자 동선 개선
  - 총운 메뉴 클릭 시 저장 리포트가 있으면 최신 리포트 상세로 이동
  - 없으면 입력 화면으로 이동

## API/배포 안정화
- Vercel 환경용 API 추가
  - `api/orrery/saju.js` 신설
- 프론트 API 호출 에러 처리 강화
  - status/content-type 체크 후 JSON 파싱
- 배포 누락 방지
  - 루트 `public` ↔ `fate-feeling/public` 동기화 루틴 적용

## 오늘의 비책
- 이미지 저장 플로우 안정화
  - html2canvas 호환/렌더링 오류 대응
  - 앱 내 미리보기 팝업은 제거, 다운로드 단일 흐름으로 단순화
- 데이터 기반 하이라이트 추가
  - 운명 지수 90점 이상일 때 하단 `비책` 아이콘 special glow

## AI 상담 UX 개선
- 입력 위계 개편
  - 메인: 고민 내용
  - 보조: 상담 목적(선택)
- 조언 스타일 칩 추가
  - #현실조언 #따뜻한공감 #명쾌한해결 #심리분석
- 모바일 타이포/헤더 정렬 보정
  - 폰트 과대/밀림 문제 지속 보정
- 이번 핫픽스
  - 고민 내용 입력칸 크기 축소 (`rows=4`, min/max height 축소)

## 체크리스트(재발 방지)
- 닫기 버튼 동작은 배포 전 **열기→닫기 2회 반복** 필수 확인
- 모바일 기준으로 라우트별 상단바/하단바 겹침 여부 확인
- API 라우트 실서버 호출(200 + JSON) 확인
